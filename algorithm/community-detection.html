    Community Detection¶

    In graph theory, community detection is the task of identifying groups of nodes in a graph that are densely connected internally. These groups are called communities, and the goal is to maximize the number of edges within a community and minimize the number of edges between communities. Or, in other words, community detection is used to find clusters of related data points. In the case of flow cytometry data, for example, communities could represent a cluster of similar cells, such as B cells.

    Before discussing community detection algorithms, it is important to understand some basic graph terminology.

    Graphs¶

    A graph is composed of nodes (vertices) and edges. Each edge is connected to two nodes, and each node may be connected to zero or more edges. When drawn in a diagram, edges are typically represented by straight lines, while nodes are represented by circles or points.

    In set notation:

    \[G = (V, E)\]

    where \(V\) is the set of nodes, and \(E\) is the set of edges:

    \[ \begin{align}\begin{aligned}V = \{v_1, v_2, \ldots, v_n\}\\E = \{e_1, e_2, \ldots, e_m\}\end{aligned}\end{align} \]

    We could alternatively write the edges with respect to the nodes they connect:

    \[\begin{split}E &= \{(v_i, v_j), (v_k, v_l), \ldots\} \\ &= \{e_{ij}, e_{kl}, \ldots\}\end{split}\]

    where \(e_{ij}\) is a directed edge from \(v_i\) to \(v_j\).

    Communities¶

    A community is a unique set of nodes:

    \[\begin{split}C_i &= \{v_1, v_2, \ldots, v_k\} \\ C_i &\subseteq V \\ C_i &\bigcap C_j = \emptyset ~ \forall ~ i \neq j\end{split}\]

    and the union of all communities must be the total set of nodes:

    \[\bigcup_{i=1}^{n} C_i = V\]

    Ideally, when finding communities (or clusters), we want to maximize the number of edges within a community and minimize the number of edges between communities.

    Partitions¶

    A partition is the set of all communities:

    \[\mathcal{P} = \{C_1, C_2, \ldots, C_n\}\]

    where each community is unique.

    Modularity¶

    How can we assess if our partition is “good”, i.e. that we have clustered the data well? One way is to use modularity. Modularity is a measure of the quality of a partition, and it is defined as:

    \[Q = \frac{1}{2M} \sum_{\mu=1}^N \sum_{\nu=1}^N \left(A_{\mu\nu} - E[n_{\mu}, n_{\nu}] \right) \delta(c_{\mu}, c_{\nu})\]

    where:

    • \(M\) is the total number of edges in the graph

    • \(N\) is the number of nodes in the graph

    • \(n_{\mu}\) is node \(\mu\)

    • \(c_{\mu}\) is the community to which node \(n_{\mu}\) belongs

    • \(A_{\mu\nu}\) is the adjacency matrix of the graph:

      \[\begin{split}A_{\mu\nu} := \begin{cases} 1 & \text{ if } n_{\mu} \text{ and } n_{\nu} \text{ are connected} \\ 0 & \text{ otherwise} \end{cases}\end{split}\]

    • \(E[n_{\mu}, n_{\nu}]\) is the probability that the nodes \(n_{\mu}\) and \(n_{\nu}\) would be connected if the edges were placed at random

    • \(\delta(c_{\mu}, c_{\nu})\) is the Kronecker delta function:

      \[\begin{split}\delta(c_{\mu}, c_{\nu}) := \begin{cases} 1 & \text{ if } c_{\mu} = c_{\nu} \\ 0 & \text{ otherwise} \end{cases}\end{split}\]

    The modularity ranges from -1 to 1, where 1 indicates a perfect partition and -1 indicates a partition that is no better than random.

    A common way to measure \(E[n_{\mu}, n_{\nu}]\) is:

    \[E[n_{\mu}, n_{\nu}] = \frac{k_{\mu} k_{\nu}}{2M}\]

    where:

    • \(k_{\mu}\) is the degree of node \(n_{\mu}\), i.e. the number of edges connected to node \(n_{\mu}\)

    Example 2: Nodes with High Likelihood of Connection¶

    Consider the following graph:

    Here the nodes \(n_1\) and \(n_5\) are in the same community. Let’s compute the probability that they would be connected by chance:

    \[E[n_1, n_5] = \dfrac{k_1 k_5}{2M} = \dfrac{4 \cdot 3}{2 \cdot 8} = \dfrac{12}{16} = 0.75\]

    So, it is quite likely that these nodes would have been connected by chance, because they are both highly-connected nodes.

    The adjacency matrix gives us:

    \[A(n_1, n_5) = 1\]

    Hence, the modularity for this pair of nodes is:

    \[Q = \dfrac{1}{2M} \left(1 - 0.75\right) = \dfrac{1}{16} = 0.0625\]

    Consider the following graph:

    Here the nodes \(n_2\) and \(n_6\) are in the same community, but they aren’t connected to each other. Let’s compute the probability that they would be connected by chance:

    \[E[n_2, n_6] = \dfrac{k_2 k_6}{2M} = \dfrac{2 \cdot 3}{2 \cdot 8} = \dfrac{6}{16} = 0.375\]

    The adjacency matrix gives us:

    \[A(n_2, n_6) = 0\]

    Hence, the modularity for this pair of nodes is:

    \[Q = \dfrac{1}{2M} \left(0 - 0.375\right) = -\dfrac{3}{16} = -0.1875\]

    So, because these nodes are not connected but in the same community, the modularity is negative.
